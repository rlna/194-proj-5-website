Project 5: [Auto] Stitching Photo Mosiacs

Fall 2020

Overview

Part A.1: Shoot the Pictures

I took photos to eventually align. For the purposes of alignment, I made sure each photo overlapped with each other by about 40%.

Part A.2: Recover Homographies

To recover homographies, I chose 6 corresponding points on each image.

Part A.3: Warp the Images

I warped the images to each other using the homography matrix recovered in the previous section. In order to later overlay the images, I first placed the images in a bounding box constructed by finding the new coordinates of each warped corner. When warping, I composed the homography matrix with a translation matrix to account for the vertical and horizontal shifts. The image on the left is the warped image.

Part A.4: Image Rectification

To rectify images, I found a homography matrix using points I wanted to rectify, and the corner points of a frontward facing box. My rectified images are below.

Part A.5: Blend Images into a Mosaic

Since the images were already in equally-sized arrays and in corresponding positions, after adjusting their bounding boxes in part 3, I simply took the max value at each point between the two images to blend them together. There was some aliasing in this method, but only because my friend's position altered the background where the photo would have been stitched together. Either way his half-disappearance made for a cool effect. After warping the first image to the second and blending them together, I warped the third image to the blended image and added it as well.

Part A.6: What I've Learned

The coolest thing I learned was how to warp any image to make it look like you are viewing it from a different angle. Using a homography matrix, I learned how to manipulate photos to 'step inside' them, even on a superficial, 2d basis.
